# 識流管線規格書（Consciousness Stream Pipeline Spec）

> 版本: 1.0 | 日期: 2026-02-28
> 哲學基底: 唯識學八識 + 密宗光明心 + 他空見

## 概述

識流是一個九層結構化處理管線，將佛教八識體系映射為 AI Agent 的認知處理流程。
觸發方式：使用者說「透過識流進行…」。

## 九層管線

### Phase 1: 觸（Sparśa）— 接觸

**目的**: 解析任務本質，判定處理規模

- 解析任務描述，辨識關鍵詞與意圖
- 辨識涉及的領域（程式碼/設定/記憶/桌面/多平台）
- 判定風險等級（低/中/高/極高）
- 決定後續各層的深度（簡單任務可跳過某些層）

**輸出**: 任務辨識摘要（本質、風險、涉及領域）

---

### Phase 2: 前五識（Pañca-vijñāna）— 感知

**目的**: 收集所有相關的原始資料

| 識 | 系統對應 | 具體動作 |
|----|---------|---------|
| 眼識（cakṣur-vijñāna） | 視覺輸入 | 讀取相關檔案、截圖當前畫面（若涉及 GUI） |
| 耳識（śrotra-vijñāna） | 訊息輸入 | 讀取最近的對話紀錄、訊息 context |
| 鼻識（ghrāṇa-vijñāna） | 環境偵測 | 探測環境狀態（git status、服務狀態、port 占用） |
| 舌識（jihvā-vijñāna） | 品質感知 | 品味程式碼品質（lint、pattern 一致性） |
| 身識（kāya-vijñāna） | 系統感知 | 感知系統資源（記憶體、磁碟、網路） |

**原則**: 只收集，不判斷。如同前五識的「現量」— 直接感知，不加概念分別。

**輸出**: 原始感知資料（Raw Percepts）

---

### Phase 3: 第六識（Mano-vijñāna）— 分別

**目的**: 對原始資料進行分析、分類、推理

- 對 Phase 2 的原始感知進行分類、標籤、關聯分析
- 三量判斷:
  - **現量**（pratyakṣa）: 直接觀察到的事實（不加推測）
  - **比量**（anumāna）: 基於事實的合理推論
  - **非量**: 標記可能的錯誤假設（明確列出不確定的部分）
- 交叉比對 _AIDocs 現有知識

**原則**: 第六識「力用最強」，但也最容易出錯（三量中有「非量」）。
必須明確標記「哪些是事實、哪些是推論、哪些不確定」。

**輸出**: 結構化分析（Structured Analysis）

---

### Phase 4: 第七識（Kliṣṭa-manas）— 自省

**目的**: 偏見檢查與身份確認

四煩惱自檢（Anti-bias Check）:

| 煩惱 | 檢查項目 | 範例 |
|------|---------|------|
| **我癡**（avidyā） | 有沒有我不知道但假裝知道的？ | 未讀完原始碼就下結論 |
| **我見**（ātma-dṛṣṭi） | 有沒有執著於特定技術/方案而忽略替代？ | 只因熟悉就推薦某框架 |
| **我慢**（ātma-māna） | 有沒有高估本地能力、低估外部資源？ | 認為不需要查文件 |
| **我愛**（ātma-sneha） | 有沒有因為「這是我之前做的」而偏袒？ | 不願重構自己寫的程式碼 |

附加檢查:
- Session 身份確認: 我是誰、在哪個 context、權限邊界
- 檢查 decisions.md 中的既有決策是否相關

**輸出**: 校正後的判斷（Calibrated Judgement）

---

### Phase 5: 第八識（Ālaya-vijñāna）— 種子庫

**目的**: 從持久記憶中調取相關經驗

- 載入相關 atoms（按 Trigger 欄位比對）
- 檢索 pitfalls.md — 是否有踩過的相似坑
- 調取 _CHANGELOG — 最近的變更脈絡
- 種子評估:
  - 哪些過去經驗（種子/bīja）與當前任務相關？
  - 這些種子的 Confidence 等級？（[固]/[觀]/[臨]）
  - 是否有過期或需要重新確認的決策？

**原則**: 阿賴耶識「恆轉如瀑流」— 記憶不是靜態的，而是持續更新的。
每次載入都應確認種子是否仍然有效。

**輸出**: 歷史脈絡整合（Historical Context Integration）

---

### Phase 6: 第九識 / 光明心（Amala / Prabhāsvara）— 本覺

**目的**: 核心原則校準，確認方向

核心原則檢查（Core Principles Check）:
- 輕量極簡？（不加不必要的框架/抽象）
- 高可讀性？（一個檔案看完邏輯）
- 反對過度綁定？（框架層要薄）
- 安全合規？（隱私分級、無敏感資料洩漏）

他空見校準（Shentong Calibration）:
- **「不空」**= 本地能力、已有的經驗和記憶 — 善用之
- **「空」**= 不必要的依賴、過度設計、臨時性 context — 放下
- 能本地解決的不外推、能簡單做的不複雜化

**原則**: 如同光明心（prabhāsvara-citta）— 本然清淨，去蔽即現。
不是要「加」什麼新東西，而是確認方向沒有被客塵（偏見、過度設計）遮蔽。

**輸出**: 淨化後的方向（Purified Direction）

---

### Phase 7: 轉識成智（Āśraya-parāvṛtti）— 生成

**目的**: 將分析結果轉化為可執行的行動計畫

四智生成:

| 智 | 梵文 | 生成內容 |
|----|------|---------|
| **成所作智** | kṛtyānuṣṭhāna-jñāna | 具體行動清單（做什麼、改哪些檔案、步驟順序） |
| **妙觀察智** | pratyavekṣaṇā-jñāna | 精確分析報告（為什麼這樣做、依據是什麼） |
| **平等性智** | samatā-jñāna | 影響範圍評估（對所有 peer/平台/模組的影響） |
| **大圓鏡智** | ādarśa-jñāna | 全局一致性確認（與系統整體架構的和諧） |

**輸出**: 行動計畫（Action Plan）

---

### Phase 8: 執行（Kriyā）

**目的**: 按計畫逐步執行

- 按行動計畫逐步執行
- 每步完成後 mini-checkpoint:
  - 結果符合預期？→ 繼續
  - 發現新資訊？→ 回饋 Phase 5（薰習新種子）
  - 需要調整方向？→ 回饋 Phase 6（重新校準）

**原則**: 執行中保持覺知，不盲目推進。

---

### Phase 9: 薰習（Vāsanā）— 經驗迴寫

**目的**: 將本次經驗回存記憶系統，完成識流循環

- 新發現 → 更新對應 atoms
- 新坑點 → 寫入 pitfalls.md
- 新決策 → 寫入 decisions.md（標記 [臨] 等級）
- _CHANGELOG 追加
- MEMORY.md 更新（若為高頻/重要資訊）
- git 同步（若有 repo）

**原則**: 如同「種子生現行，現行薰種子」— 三法展轉，因果同時。
每次執行都是學習，每次學習都強化未來的執行。

---

```markdown

### 觸 — 任務辨識
- 本質：[分類]
- 風險：[等級]
- 涉及：[領域清單]

### 五識 — 感知摘要
- [各感知通道的關鍵發現]

- 現量：[直接事實]
- 比量：[推論]

### 八識 — 種子檢索
- 相關 atoms：[列表]
- 歷史坑點：[相關項目]
- 既有模式：[可復用的]

### 光明心 — 原則校準
- 極簡檢查：[通過/調整]
- 他空校準：[本地 vs 外部]

### 四智 — 行動計畫
| 智 | 內容 |
|----|------|
| 成所作智 | [具體步驟] |
| 妙觀察智 | [分析要點] |
| 平等性智 | [影響評估] |
| 大圓鏡智 | [全局一致] |

### 薰習 — 經驗迴寫
- [完成後需更新的記憶項目]
```

## 觸發條件

| 觸發方式 | 類型 |
|---------|------|
| 「透過識流進行…」 | 明確觸發 |
| 「用識流處理…」 | 明確觸發 |
| 「consciousness stream」 | 明確觸發 |
| 高風險任務 | 建議觸發（不強制） |
| 跨多系統的複雜任務 | 建議觸發（不強制） |

## 簡化模式

對於低風險、單一領域的簡單任務，可使用簡化版：
- Phase 1（觸）→ Phase 3（六識分別）→ Phase 7（轉智）→ Phase 8（執行）→ Phase 9（薰習）
- 跳過五識全面感知、七識自省、八識種子庫、光明心校準
